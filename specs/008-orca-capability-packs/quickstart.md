# Quickstart: Orca Capability Packs

## Goal

Validate that Orca can describe optional workflow behavior through explicit
pack boundaries without recreating trait complexity.

## Scenario 1: Define a pack

1. Define one realistic capability pack.
2. Verify its purpose, affected commands, prerequisites, and activation mode
   are explicit.

## Scenario 2: Core remains understandable

1. Inspect the core command set alongside the pack definitions.
2. Verify the core workflow still makes sense without hidden pack knowledge.

## Scenario 3: Downstream behavior is not treated as foundation

1. Inspect the `yolo` pack role.
2. Verify it is classified as downstream rather than foundational.
